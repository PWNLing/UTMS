# UTMS 第三阶段：实时航迹、历史回放与电子围栏告警需求文档

> 文档状态：已确认，待实现
> 依赖：`docs/PRD.md`（第一阶段基线）、`docs/PRD-phase2.md`（第二阶段既有能力）、`docs/code-standards.md`、`CONTEXT.md`

---

## 1. 背景与目标

第一阶段定义的雷达帧仍然是完整的当前帧快照：每一帧替换上一帧，消失目标立即从当前画面移除。第三阶段不改变该语义，而是在每个**已接受的合法 `RadarFrame`**之后扩展三条彼此独立的数据链路：

```text
已接受的 RadarFrame
    ├── 实时显示：当前帧地图、表格、统计、实时短航迹
    ├── 历史记录：异步 SQLite 持久化、查询、CSV 导出、回放
    └── 告警分析：异步电子围栏与规则评估、告警持久化
```

本阶段目标是：

1. 在实时地图中呈现短时目标航迹，而不改变当前帧目标集合；
2. 保存历史雷达场景，按条件查询并回放；
3. 支持圆形、矩形和多边形电子围栏；
4. 针对围栏事件和围栏内超速生成、处置、查询告警；
5. 任何历史库或告警子系统问题都不得阻塞 UDP 接收和实时显控。

## 2. 已确认范围

### 2.1 保持的基线

- 仅支持单雷达；雷达坐标继续为 GCJ-02；
- UDP 的完整帧判定、单目标容错、序号接受策略和当前帧替换语义保持不变；
- 视频、录像、YOLO、登录和系统监控的第二阶段需求不因本阶段改变；
- 在线和离线地图继续共用同一个 `MapPanel` 业务状态。

### 2.2 暂不纳入范围

- 多雷达数据融合、跨雷达航迹关联；
- 视频回放、录像查询、告警触发录像或雷达—视频关联；
- 云端同步、远程数据库、数据库备份服务；
- 任意脚本告警规则、机器学习告警或通知外发；
- 围栏旋转、圆弧多边形、三维围栏；
- 历史轨迹插值、补点或位置预测；
- 自动化 UI 测试。

## 3. 总体架构与线程边界

### 3.1 独立数据链路

- `MainWindow` 的实时显示路径接收所有已接受帧，保持当前帧语义；
- `HistoryRecorder` 在独立工作线程按采样策略将历史帧批量写入 SQLite；
- `AlertEngine` 在独立工作线程评估实时帧，并只发出结构化 `AlertEvent`；
- 工作线程不得访问 `QWidget`；所有界面更新经 queued signal 回到 GUI 线程；
- 历史记录或告警分析的失败只影响各自状态，绝不停止 UDP 接收、实时地图、表格或统计。

### 3.2 地图状态边界

`MapPanel` 是在线与离线地图共同的状态边界，统一持有并下发：

- 当前实时帧或当前回放帧；
- 地图中心、缩放、目标选中和高亮；
- 实时短航迹、历史回放轨迹；
- 围栏和告警位置标记。

切换地图模式时，不得重置上述状态。

## 4. 实时短航迹

### 4.1 采样与保留

- 每个当前目标固定每 `500 ms` 最多采样一个位置；
- 默认保留最近 `30 秒`，可在本次运行内切换为关闭、10 秒、30 秒或 1 分钟；
- 默认只显示选中目标；提供“显示所有目标航迹”开关；
- 轨迹显示配置仅属于本次运行，不持久化；
- 当前帧中目标消失时，目标图标立即移除，航迹线继续保留 5 秒；同一 `track_id` 在此窗口内重新出现可续接。

### 4.2 断段规则

同一 `track_id` 不代表永久连续轨迹。必须开始新的轨迹段而不得连线的情况：

- 相邻采样点的时间间隔超过 3 秒；
- 相邻位置跳变超过 200 米；
- 目标消失超过 5 秒后再次出现。

### 4.3 视觉规则

- 使用按目标类别着色的实线折线、圆角连接；
- 普通目标线宽 2 px，选中目标线宽 3 px；
- 默认不显示采样点；
- 显示层每 10 个采样点分为一个线段，相邻段必须共享端点；从最新到最旧逐段降低透明度；
- 当前目标点仍按既有圆点样式显示。

## 5. 历史记录与本地存储

### 5.1 记录资格与频率

- 仅记录已接受的合法 `RadarFrame`；非法 JSON、重复帧、乱序拒绝帧及被解析器跳过的无效目标均不得写入历史；
- 实时显示仍处理每个已接受帧；历史记录默认每 500 ms 保存最新完整快照（2 FPS）；
- 操作员可选择 1 FPS、2 FPS、5 FPS 或全帧记录；选择会跨重启保留。

### 5.2 历史会话

- 每次 UDP 成功开始监听时创建一个历史会话；
- 用户停止监听时正常关闭会话；
- 应用下次启动时，仍标记为进行中的旧会话必须标为“异常结束”，并保留已成功写入的数据；
- 会话是历史查询、回放和受控删除的最小归属单位。

### 5.3 SQLite 边界

- 使用应用本地 SQLite 数据库存储历史会话、采样帧、有效目标、围栏、规则和告警；
- 历史结构为“会话 → 历史帧 → 有效目标”；
- 必须为会话 ID、历史时间、航迹 ID、目标类别建立适合查询的索引；
- 写入在工作线程中使用批量事务，禁止 GUI 线程逐目标执行 SQL；
- 历史采样频率、保留天数、围栏与规则须跨重启保留。

### 5.4 失败与恢复

- SQLite 打开、事务或写入失败时，历史状态显示“记录失败”并记录低频错误日志；
- 失败不得影响实时显示或告警引擎；
- 记录器每 5 秒后台尝试恢复；恢复后仅记录后续新帧，不补写失败期间的数据；
- 不使用阻塞式错误弹窗。

### 5.5 保留与删除

- 默认保留最近 7 天；可配置范围为 1–30 天；
- 启动时及每日定时清理过期历史；
- 支持显示数据库占用空间；
- 允许手动删除指定会话，删除前必须二次确认；
- 不提供云端同步、远程数据库或永久归档。

## 6. 历史查询、回放与导出

### 6.1 查询界面

右侧 `QTabWidget` 新增“历史回放”Tab。该 Tab 提供查询、导出和播放控制；查询结果复用主工作区的大表格区域。

查询条件：

- 时间范围；
- 历史会话；
- 航迹 ID；
- 目标类别。

### 6.2 回放模式

- 主窗口必须明显区分“实时模式”和“历史回放模式”；底部状态也须显示回放状态；
- 进入回放后，地图、表格、统计显示历史帧；后台 UDP 接收、历史写入与实时告警继续运行，但不得覆盖回放画面；
- 返回实时后停止播放、清除回放专用轨迹，并立即显示最新已接收实时帧；
- 回放模式下选中目标时，地图显示该目标从当前查询起点到当前回放时刻的完整历史轨迹；未选中目标不绘制长历史轨迹。

### 6.3 播放控制

- 提供播放、暂停、上一帧、下一帧、时间轴拖动，以及 0.5×、1×、2×、4×速度；
- 拖动时间轴时暂停播放并加载最接近选定时刻的历史帧；
- 相邻历史帧时间间隔超过 5 秒时，直接跳到下一帧并显示“数据中断 N 秒”；不得插值或虚构目标位置。

### 6.4 导出

- 仅支持 CSV；不提供历史帧 JSON 导出；
- CSV 导出当前查询结果或单条选定航迹的轨迹点；
- CSV 至少包含：会话 ID、帧时间、接收时间、序列号、航迹 ID、类别、纬度、经度、速度、距离；
- 只导出当前筛选范围内的数据。

第一阶段要求 `altitude` 不进入业务模型；第三阶段沿用该约束，不存储或导出发送端的海拔字段。

## 7. 电子围栏

### 7.1 围栏形状与坐标

所有围栏使用 GCJ-02，并支持：

- 圆形：中心点和半径；
- 矩形：与经纬线平行，由西南角和东北角定义，不支持旋转；
- 多边形：3–20 个顶点，禁止自相交。

### 7.2 管理与编辑

围栏必须支持新建、编辑、删除、重命名、定位、启用、禁用、显示和隐藏。

- 圆形可调整中心和半径；
- 矩形可拖动四角或整体移动；
- 多边形可拖动顶点或整体移动；
- 禁用围栏使用灰色虚线显示（或由显示开关隐藏），且不参与新告警判断；
- 禁用或删除围栏都不得删除已经产生的告警历史。

## 8. 告警规则与状态机

### 8.1 固定规则类型

首版仅提供以下规则，且每条规则均关联一个围栏：

1. 稳定进入围栏；
2. 稳定离开围栏；
3. 围栏内停留超时；
4. 围栏内超速。

不提供全局超速和任意脚本规则。

### 8.2 规则配置

每条规则至少包含名称、规则类型、关联围栏、适用目标类别（可多选）、告警等级、阈值、确认时间、冷却时间、启用状态和备注。

- 告警等级：提示、警告、严重；
- 超速阈值单位为 m/s，沿用 `velocity_mps`，不换算为 km/h；
- 停留阈值范围为 5 秒–24 小时；
- 进入、离开和超速规则的确认时间范围为 0–60 秒，默认 1 秒；0 秒表示不额外等待；
- 停留规则仅由其停留阈值决定，不额外叠加确认时间；
- 冷却时间范围为 0 秒–24 小时，默认 30 秒；0 秒表示只依赖条件恢复后的去重规则。

### 8.3 边界防抖与目标消失

- 进入使用原始围栏边界；
- 已确认在围栏内的目标必须到边界外至少 5 米，才满足离开候选条件；
- 进入、离开和超速必须连续满足各自确认时间才改变状态或报警；
- 目标临时消失时，围栏状态最多保留 3 秒；3 秒内重现则延续，超过 3 秒仅清理状态，不生成离开告警。

### 8.4 去重与冷却

- 同一规则与同一目标在条件持续成立期间只产生一次告警；
- 条件必须先恢复正常，才允许再次进入可告警状态；
- 再次触发仍受该规则冷却时间限制。

## 9. 告警中心与历史

### 9.1 告警中心

新增“告警中心”Tab，至少包括：

- 未确认告警数量；
- 单条确认、全部确认、处理备注、CSV 导出和规则管理入口；
- 时间、等级、规则、航迹 ID、类别、确认状态和描述组成的告警表格；
- 围栏管理入口。

每个告警保存发生时间、规则、等级、航迹 ID、类别、位置、速度、距离、围栏、描述、确认状态、确认时间、确认人和处理备注。当前固定登录用户下，确认人记录为 `root`。

### 9.2 告警提示与联动

- 底部状态栏显示未确认告警数量；
- 严重告警播放一次提示音、短暂高亮地图目标，并显示右下角非模态通知；
- 不使用每帧弹窗或阻塞式 `QMessageBox`；
- 点击告警时定位到告警位置；当前目标仍存在则同时选中地图和表格目标，已消失则显示历史位置标记及完整详情。

### 9.3 告警历史与回放

- 告警确认只改变处理状态，绝不删除告警；
- 告警历史可按时间、等级、规则、围栏、航迹 ID、目标类别和确认状态查询，并支持 CSV 导出；
- 历史回放只读取并显示当时已持久化的告警标记，绝不重新执行规则或生成新告警。

## 10. 日志与错误可见性

- 记录历史会话创建、关闭、异常结束、持久化失败、恢复和清理；
- 记录围栏和规则的创建、修改、启停及删除；
- 记录告警引擎启动、停止、严重评估或持久化失败，以及告警创建与确认；
- 不对每个正常历史帧、每次正常围栏判断、每个正常轨迹采样点或每个正常告警循环写高频日志；
- 历史和告警错误须在相应 UI 中可见，但不得用阻塞弹窗破坏操作。

## 11. 验收要求

### 11.1 实时航迹

1. 选中目标后可显示最近 30 秒、500 ms 采样的实时短航迹；可切换关闭、10 秒、30 秒、1 分钟和全部航迹显示。
2. 目标暂时消失不保留目标图标；5 秒内重现可续接，超过 5 秒、间隔超过 3 秒或跳变超过 200 米时不连线。
3. 航迹按类别着色、分段渐隐，选中目标与普通目标线宽不同。
4. 在线/离线地图切换后实时航迹与选中状态不丢失。

### 11.2 历史记录与回放

1. 监听启动/停止分别创建/关闭历史会话；异常退出后的未关闭会话下次启动被标记为异常结束。
2. 可配置 1、2、5 FPS 或全帧记录，默认 2 FPS；实时显示仍处理全部接受帧。
3. 可按时间、会话、航迹 ID、类别查询，按查询结果导出 CSV。
4. 可播放、暂停、逐帧、拖动时间轴并切换规定速度；长缺口不插值且明确提示。
5. 回放中实时帧继续接收、记录和分析，但不会覆盖回放画面；返回实时后立即显示最新实时帧。
6. SQLite 故障不会使 UDP 或告警停止；恢复后能继续记录新帧。
7. 保留期、受控删除、数据库占用显示符合规定。

### 11.3 围栏与告警

1. 可创建、编辑和持久化圆形、非旋转矩形及合法多边形围栏；非法自相交或超顶点数多边形被拒绝。
2. 围栏启停、显示隐藏和地图模式切换均符合规定；禁用围栏不删除历史告警。
3. 可创建四种规定规则；围栏内超速只以 m/s 阈值工作。
4. 边缘抖动、短时条件波动和目标短时消失不会造成重复或错误的进入/离开告警。
5. 持续条件只报警一次，恢复且冷却期满足后才可再次报警。
6. 告警可查询、CSV 导出、单条/批量确认和填写备注；确认后仍可追溯。
7. 严重告警使用非模态提示；点击告警可定位位置。
8. 回放不重新执行规则或新增告警；在线/离线地图显示一致的围栏与告警状态。

### 11.4 自动化测试与稳定性

Qt Test 至少覆盖不依赖实际地图和 UDP 的：轨迹采样/断段、历史会话状态、采样策略、SQLite 查询与失败降级、时间轴选帧、围栏几何判定、规则确认/迟滞/目标消失、去重冷却和回放不重算告警。

在 Windows 10/11 真实运行环境中人工验证在线/离线地图渲染、围栏编辑、CSV 内容、回放联动和告警提示；连续运行两小时不得出现崩溃、明显持续内存增长或严重界面卡顿。
