## 绿色发布目录

- [ ] 在未配置 Qt、FFmpeg 和 ONNX Runtime 开发环境的 Windows 10/11 64 位机器上，可从 `bin/UTMS.exe` 启动。
- [ ] `bin/` 包含 Qt 与 MSVC 运行库，以及 `avcodec-*`、`avformat-*`、`avutil-*`、`swscale-*` 等 FFmpeg DLL。
- [ ] `bin/onnxruntime.dll` 存在；ONNX Runtime 包提供的其他 DLL 也已随包复制。
- [ ] `bin/models/yolo26/` 包含 `yolo26n.onnx`、`classes.txt` 和 `model.json`。
- [ ] 发布目录的 `third_party/` 下没有模型资源；模型只位于 `bin/models/`。
- [ ] `licenses/ffmpeg/` 和 `licenses/onnxruntime/` 包含随第三方二进制交付的许可与声明文件。
- [ ] 发布包不包含真实 `bin/config/amap.json`；部署人员只在验收副本中按需配置 Key。

## 启动登录

- [ ] 启动 `bin/UTMS.exe` 后首先显示“GUET-UTMS 用户登录”，登录前不显示主窗口，也不启动 UDP、RTSP 或系统监控工作线程。
- [ ] 登录界面显示 GUET Logo 和规定的左右布局；缩放窗口和调整 Windows DPI 后，Logo 保持比例且输入控件可正常使用。
- [ ] 账号默认填写 `root`，密码默认留空并掩码显示；使用 `root / 123456` 点击“登录”或按 Enter 均可进入主窗口。
- [ ] 账号或密码为空时显示“请输入账号和密码”；凭据错误时显示“账号或密码错误”。失败后密码被清空且密码框重新获得焦点。
- [ ] 取消或关闭登录对话框后应用直接退出，主窗口不短暂出现；日志和本机文件中不包含输入的密码或持久化登录状态。

## 雷达基线回归

- [ ] 初始 UDP 状态为红色且端口为 `10000`；启动监听后变黄，合法帧到达后变绿，停止后恢复红色。
- [ ] 在线和离线地图、表格筛选排序、地图表格联动、统计图和底部数量仍按当前帧同步更新。
- [ ] 非法 JSON 保留上一帧；重复包和乱序包不使界面回退。

## RTSP 与检测功能

- [ ] 应用启动后不自动连接；视频 Tab 显示可编辑的默认 RTSP 地址。
- [ ] 真实 RTSP 通过 TCP 连接，界面依次显示连接中和播放中，画面保持比例且检测默认关闭。
- [ ] 开启检测后出现检测框、中文类别与置信度；行人、自行车、汽车、卡车以外的类别显示为未知。
- [ ] 关闭检测后预览继续，推理停止且检测框立即清除；再次开启时无需重新加载模型。
- [ ] 主动断开后状态为已断开，最后一帧与检测框立即清除，且不再自动重连。
- [ ] 非主动断流后状态为重连中，旧画面立即清除，约每 3 秒重试；恢复后继续播放。
- [ ] 断流前检测已开启时，自动重连成功后检测自动恢复。
- [ ] 使用验收副本临时移走 `model.json` 后开启检测：显示检测不可用，RTSP 预览继续，日志记录模型资源加载失败；测试后恢复文件。
- [ ] 使用验收副本临时以无效 ONNX 文件替换模型后开启检测：显示检测不可用，RTSP 预览继续，日志记录推理初始化失败；测试后恢复模型。

## RTSP 原始录像

- [ ] RTSP 未播放、连接中、重连中或已断开时“开始录制”不可用；播放中可进入“准备录制”。
- [ ] 从点击开始后的下一个关键帧进入“录制中”，时长从 `00:00:00` 开始；10 秒无关键帧时显示超时且无空文件残留。
- [ ] 录制中按钮显示“停止录制”，重复点击不会创建第二个录像；停止后显示“保存完成（HH:mm:ss）”。
- [ ] `bin/recordings/` 按需创建；连续两次在同一秒开始时使用 `_01` 等后缀，已有文件不被覆盖。
- [ ] 用 `ffprobe` 检查成品：仅包含一条 H.264/H.265 视频轨，无音频、检测框或界面叠加，时长和文件大小有效。
- [ ] 录制中中断 RTSP 源：当前 MP4 安全封尾并可播放；恢复源后自动重连，但录像保持关闭且不自动创建新文件。
- [ ] 录制中主动断开和关闭应用均能保存可播放文件；录像失败不影响预览、检测或后续重连。

开发环境可使用 `UTMS_rtsp_recording_acceptance` 配合本地 MediaMTX/FFmpeg 测试源；工具退出码为 0，且输出 `RECORDING_VERIFIED=1`。正式验收仍需在绿色发布目录中人工操作并保存 `ffprobe` 输出作为证据。

## 系统监控

- [ ] “系统监控”Tab 初始显示监控未开启，主机 CPU、主机内存、UTMS 进程 CPU 和 UTMS 进程内存均没有旧数值。
- [ ] 点击“开启监控”后状态和按钮文字正确切换，四项指标约每秒刷新；主机内存同时显示已用量、总量和使用率。
- [ ] 点击“停止监控”后停止采样，状态和按钮恢复，四项旧数值立即清空且不会被迟到的采样结果重新填充。
- [ ] 人为制造一个可恢复的单项指标读取失败时，仅该指标显示明确错误，其他指标继续刷新，UDP、地图、RTSP、录像和 YOLO 不受影响。
- [ ] 监控开启期间关闭应用，采样线程可正常结束，界面不发生明显卡顿；重新启动后监控仍默认为关闭。

## 日志检查

- [ ] `bin/logs/utms.log` 记录 RTSP 连接、主动断开、连接失败、播放中断和重连上下文，但不记录包含用户名或密码的完整 URL。
- [ ] 日志区分模型资源加载失败、ONNX 推理初始化失败和运行中推理失败。
- [ ] 日志记录录像请求、实际开始、停止原因、输出路径、保存结果和失败清理；录像失败信息足以定位初始化、写包或封尾阶段。
- [ ] 日志记录系统监控启停和指标读取失败，但不记录每秒正常采样；登录和运行日志均不包含密码。
- [ ] 严重线程生命周期或未捕获运行错误使用 `CRITICAL` 或 `FATAL` 级别记录。
- [ ] 正常视频帧、正常录像包和正常检测结果不会逐帧刷日志；轮转后最多保留当前文件及 4 个归档文件。
