# 运行日志

UTMS 在首次启动时于本目录创建 `utms.log`。

- 单个日志文件最大 10 MB。
- 最多保留当前文件及轮转文件共 5 个。
- 日志记录程序生命周期、登录取消、UDP 启停与异常、报文校验、地图配置和瓦片缺失等事件。
- 视频日志记录 RTSP 连接、主动断开、失败、播放中断和重连；URL 只保留协议、主机和端口，不写入凭据。
- 检测日志区分模型资源加载失败、ONNX 推理初始化失败和运行中推理失败。
- 录像日志记录开始请求、等待关键帧、实际开始写入、停止原因、输出路径、保存结果，以及初始化、写包、封尾、关闭和无效文件清理失败。
- 系统监控日志记录监控启停、采样线程生命周期异常和单项指标读取失败。
- 历史日志记录会话创建、关闭、异常结束、数据库故障、后台恢复、过期清理和受控删除。
- 围栏与告警日志记录围栏和规则的创建、修改、启停、删除，以及告警生成、确认和持久化失败。
- 严重线程生命周期或未捕获运行错误使用 `CRITICAL` 或 `FATAL` 级别记录。
- 日志不记录密码，也不会为正常接收的视频帧、正常录像包、正常检测结果、历史采样点、围栏判断、告警循环或每秒成功的系统指标采样写入高频记录。

排查故障时，请保留本目录中的全部 `utms*.log` 文件。
