# 运行日志

UTMS 在首次启动时于本目录创建 `utms.log`。

- 单个日志文件最大 10 MB。
- 最多保留当前文件及轮转文件共 5 个。
- 日志记录程序生命周期、UDP 启停与异常、报文校验、地图配置和瓦片缺失等事件。
- 视频日志记录 RTSP 连接、主动断开、失败、播放中断和重连；URL 只保留协议、主机和端口，不写入凭据。
- 检测日志区分模型资源加载失败、ONNX 推理初始化失败和运行中推理失败。
- 严重线程生命周期或未捕获运行错误使用 `CRITICAL` 或 `FATAL` 级别记录。
- 正常接收的视频帧和正常检测结果不会逐帧写入日志。

排查故障时，请保留本目录中的全部 `utms*.log` 文件。
